## 1.6 Funding/Support Buttons

The Support section of the profile README presents two call-to-action buttons that enable visitors to contribute financially to the maintainer. These buttons leverage third-party platforms (“Buy Me a Coffee” and “Ko-fi”) by embedding their respective badges as linked images.

### Markup Structure

The HTML snippet defining the Support buttons is as follows :

```html
<h3 align="left">Support:</h3>
<p>
  <a href="https://www.buymeacoffee.com/https://paypal.me/roberthPendringuez">
    <img align="left"
         src="https://cdn.buymeacoffee.com/buttons/v2/default-yellow.png"
         height="50"
         width="210"
         alt="https://paypal.me/roberthPendringuez" />
  </a>
  <a href="https://ko-fi.com/kristinedais10@gmail.com">
    <img align="left"
         src="https://cdn.ko-fi.com/cdn/kofi3.png?v=3"
         height="50"
         width="210"
         alt="kristinedais10@gmail.com" />
  </a>
</p>
<br><br>
```

| Ko-fi | https://ko-fi.com/kristinedais10@gmail.com | https://cdn.ko-fi.com/cdn/kofi3.png?v=3 | kristinedais10@gmail.com |


### URL Encoding and Link Correctness

- **Buy Me a Coffee button**

The `href` attribute is constructed by appending the full PayPal URL (`https://paypal.me/roberthPendringuez`) directly after the Buy Me a Coffee domain. As a result, the browser will request

`https://www.buymeacoffee.com/https://paypal.me/roberthPendringuez`

combining two protocols in one path. This concatenation is likely to produce a 404 or misdirected request, since Buy Me a Coffee expects only a username or campaign slug in the path.

- **Ko-fi button**

The `href` uses the maintainer’s email (`kristinedais10@gmail.com`) as the path segment:

`https://ko-fi.com/kristinedais10@gmail.com`

Ko-fi typically reserves the path for a username or page slug rather than an email address. This may not resolve to the intended fundraising page if the email has not been configured as a valid Ko-fi username.

By documenting the exact markup, link targets, and embedded image attributes, this section makes explicit how the funding buttons are rendered and highlights potential formatting or link-correctness issues in the current implementation.
