# Repository Purpose & How It Works (GitHub Profile README)

## 1.3 External Widgets/Embeds Inventory and Operational Risks

This section catalogs all of the externally hosted widgets and image embeds used in the `README.md`, describes their functionality, and outlines operational considerations including availability, rate limits, privacy implications, and instructions for updating parameters such as the GitHub username.

### Inventory of Dynamic Widgets

#### GitHub Readme Stats

Purpose: Displays a dynamic “stats” card (total commits, stars, forks, issues, etc.) and a “top languages” card showing the user’s most‐used languages.

Implementation: Two `<img>` tags point to the Vercel-hosted Readme Stats API:

```html
<div align="center">
  <img src="https://github-readme-stats.vercel.app/api?username=maurodesouza&hide_title=false&hide_rank=false&show_icons=true&include_all_commits=true&count_private=true&disable_animations=false&theme=dracula&locale=en&hide_border=false" alt="stats graph" height="150" />
  <img src="https://github-readme-stats.vercel.app/api/top-langs?username=maurodesouza&locale=en&hide_title=false&layout=compact&card_width=320&langs_count=5&theme=dracula&hide_border=false" alt="languages graph" height="150" />
</div>
```

Current embed parameters reference `username=maurodesouza`, which must be changed to `username=ronams03` to reflect this repository .

How to update:

- Change the `username` query parameter in both URLs to your GitHub handle (`ronams03`).
- Optionally adjust other parameters (e.g. `theme`, `hide_border`, `langs_count`).

Operational Considerations:

- Availability: Hosted on Vercel’s free tier; occasional cold starts or rate limits under heavy usage.
- Rate Limits: Unofficial API; subject to request throttling if exceeds free quota.
- Privacy: Each page view triggers an external request to `github-readme-stats.vercel.app`, exposing viewer IP and user‐agent to the service.

#### GitHub Profile Trophy

Purpose: Renders a set of “trophies” for GitHub achievements (e.g., stars, followers, commits).

Implementation: A single `<img>` tag loads from the Trophy API:

```html
<img src="https://github-profile-trophy.vercel.app/?username=maurodesouza" alt="trophy graph" height="150" />
```

Currently references `username=maurodesouza` .

How to update:

- Change `username` to `ronams03`.
- Additional query parameters (e.g. `theme`, `margin-w`) can be appended per the API docs.

Operational Considerations:

- Availability: Service hosted on Vercel; may experience downtime or slow responses.
- Rate Limits: No formal SLA; heavy traffic can lead to timeouts or blank images.
- Privacy: External image request exposes viewer metadata to `github-profile-trophy.vercel.app`.

#### GitHub Streak Stats

Purpose: Shows the current and longest GitHub contribution streak for the user.

Implementation: An `<img>` tag points to the Streak Stats endpoint:

```html
<img src="https://streak-stats.demolab.com?user=maurodesouza&locale=en&mode=daily&theme=dracula&hide_border=false&border_radius=5&order=3" alt="streak graph" height="150" />
```

References `user=maurodesouza` .

How to update:

- Change `user` to `ronams03`.
- Adjust `mode` (e.g. `weekly`), `theme`, and other query parameters as desired.

Operational Considerations:

- Availability: Unofficial service (`demolab.com`); may be slower or intermittently unavailable.
- Rate Limits: Unknown; heavy usage may trigger throttling.
- Privacy: Remote request leaks viewer IP to `streak-stats.demolab.com`.

#### Pacman Contribution Graph

Purpose: A stylized “Pacman” version of the GitHub contributions calendar, switching automatically between light and dark variants.

Implementation:

```html
<picture>
  <source media="(prefers-color-scheme: dark)" srcset="https://raw.githubusercontent.com/maurodesouza/maurodesouza/output/pacman-contribution-graph-dark.svg">
  <source media="(prefers-color-scheme: light)" srcset="https://raw.githubusercontent.com/maurodesouza/maurodesouza/output/pacman-contribution-graph.svg">
  <img alt="pacman contribution graph" src="https://raw.githubusercontent.com/maurodesouza/maurodesouza/output/pacman-contribution-graph.svg">
</picture>
```

Embedded via `<picture>` with dark/light sources .

How to update:

- Replace both `maurodesouza/maurodesouza` path segments with `ronams03/ronams03` in the URLs.
- Ensure that the corresponding SVG outputs exist under `https://raw.githubusercontent.com/ronams03/ronams03/output/`.

Operational Considerations:

- Availability: Served by GitHub’s raw content CDN; generally reliable but will 404 if the files are missing or renamed.
- Rate Limits: GitHub enforces bandwidth limits; unlikely to be reached for infrequent reads.
- Privacy: Static fetch from `raw.githubusercontent.com`; viewer metadata is logged by GitHub.

### Operational Risks Summary

- **Third-party Dependence**: All dynamic widgets rely on external hosts (Vercel, DemoLab, GitHub raw CDN). Outages or service changes can break card rendering.
- **Rate-Limiting & Throttling**: Unofficial free services may throttle requests or serve stale/failing images under high load.
- **Privacy Exposure**: Every visitor to the README triggers external image requests, exposing their IP, browser headers, and potentially geographic data to each provider.
- **Maintenance Overhead**: Manual edits required in `README.md` whenever the GitHub username or theme parameters change; mismatches lead to blank or incorrect widgets.
